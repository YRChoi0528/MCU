# stm32f10x_map.h 정리

## 역할 (한 줄 요약)
`stm32f10x_map.h`는 **STM32F10x MCU의 메모리 맵(주소 공간 배치)을 정의한 헤더**이다.

---

## 메모리 맵이란 무엇인가
메모리 맵은 다음을 의미한다.

- 어떤 주소에 무엇이 있는가
  - Flash
  - SRAM
  - 주변장치 레지스터
- CPU가 특정 주소를 읽고/쓸 때
  - 실제로 어떤 하드웨어가 반응하는가

즉,
> **주소와 하드웨어의 대응표**

이다.

---

## 왜 중요한가
MCU에서 주변장치 제어는 결국 다음 형태로 귀결된다.

```c
*(주소) = 값;
```
그 "주소"가 무엇을 의미하는지를 정의해 둔 것이</br>
바로 `stm32f10x_map.h`이다.

## 구성 요소
### 1) 메모리 영역 베이스 주소
- Flash base
- SRAM base
- Peripheral base

---

### 2) 주변장치 블록 주소
- GPIOA, GPIOB, ...
- RCC
- USART
- TIM
각 주변장치는 고정된 시작 주소를 가진다.

---

### 3) 구조체 매핑을 위한 기반
다른 헤더들(`stm32f10x_gpio.h` 등)은</br>
이 주소 위에 구조체를 씌워서 레지스터를 접근한다.

#### 예시 (개념적)
```c
#define GPIOA_BASE 0x40010800

GPIOA->ODR = 1;
```
의미:
- `0x40010800` 주소에 있는 하드웨어(GPIOA)
- 그 중 출력 데이터 레지스터를 제어
---

## 실습 관점 정리
- 해당 파일은 거의 수정하지 않는다.
- 주소 기반 하드웨어 제어의 "지도" 역할
- 레지스터 접근이 어떻게 가능한지 이해하는 데 중요
