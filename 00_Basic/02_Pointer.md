# 02. 포인터 (Pointer)

## 목적
포인터를 “주소를 담는 변수”로 이해하고, `*`(역참조), `&`(주소 연산자)를 통해
변수/배열/함수 호출에서 데이터가 어떻게 전달·변경되는지 확인한다.

## 학습 내용
- `&` : 변수의 주소를 얻는 연산
- `*p` : 포인터가 가리키는 값을 읽고/쓰는 방식(역참조)
- 포인터와 함수: 값 전달 vs 주소 전달(함수에서 원본 변경)
- 배열과 포인터의 관계(배열 이름이 식에서 포인터처럼 동작하는 이유)
- `p + 1` 같은 포인터 연산의 의미(자료형 크기 단위 이동)

## 핵심 개념 정리
- 포인터는 “값”이 아니라 “주소”를 담는다.
- `*p`는 “p가 가리키는 대상”을 의미한다.
- 함수에 포인터를 넘기면, 함수 내부에서 **원본 값을 바꿀 수 있다.**
- 배열 `a`는 식에서 보통 `&a[0]`처럼 동작한다.
