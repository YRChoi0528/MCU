# 00_Basic

## 개요

`00_Basic` 디렉터리는 **마이크로프로세서 실험 과목을 수행하기 전에 학생들이 반드시 이해하고 있어야 하는 C 언어의 핵심 개념**을 정리한 공간이다.

이 디렉터리에서 다루는 내용은 단순한 문법 수준의 C 언어(예: `for`, `while`, `if` 등)가 아니라, **MCU 실습 코드의 구조를 이해하고 수정·확장하기 위해 필수적인 언어 요소**에 초점을 둔다. 실제 실습에서는 공급사에서 제공된 코드와 헤더 파일을 분석하고 이를 기반으로 기능을 추가하거나 수정해야 하므로, 아래와 같은 C 언어 개념에 대한 이해가 선행되어야 한다.

---

## 목적

- 마이크로프로세서 실험에서 사용되는 C 코드 구조를 **읽고 이해할 수 있는 기반 지식 확보**
- MCU 주변장치 제어 코드에서 자주 등장하는 **매크로, 포인터, 구조체 기반 설계 방식에 대한 적응**
- 단순 암기가 아닌, *"이 코드가 왜 이런 형태로 작성되었는가"*를 설명할 수 있는 수준 도달

본 디렉터리는 **MCU 실습에서 제공되는 C 코드를 이해하고 수정하기 위해 필요한 핵심 언어 요소를 정리한 공간이다.

---

## 학습 내용

### 1. 매크로와 전처리기 (Preprocessor)

- `#define`을 이용한 상수 및 매크로 함수 정의
- `#if`, `#ifdef`, `#ifndef`를 이용한 조건부 컴파일
- 헤더 파일에서 전처리기가 사용되는 이유
- MCU 레지스터 정의 및 설정 코드에서 매크로가 활용되는 방식

> MCU 실습 코드에서는 **매직 넘버 대신 매크로로 정의된 값**이 빈번하게 사용되므로, 전처리기 동작에 대한 이해가 필수적이다.

---

### 2. 포인터 (Pointer)

- 포인터의 기본 개념과 메모리 주소
- 포인터와 배열의 관계
- 함수 인자로 포인터를 전달하는 이유
- MCU 실습에서 포인터가 사용되는 대표적인 사례

> 레지스터 접근, 버퍼 처리, 함수 간 데이터 공유 등에서 포인터 개념은 핵심적인 역할을 한다.

---

### 3. 함수 (Function)

- 함수 선언과 정의의 분리 (`.h` / `.c`)
- 인자 전달 방식 (값 전달, 주소 전달)
- 실습 코드에서 공통 동작을 함수로 분리하는 이유

> 대부분의 실습 예제는 **주변장치 제어를 함수 단위로 분리한 코드 구조**를 가지므로, 함수 단위 코드 해석 능력이 중요하다.

---

### 4. 구조체 (Structure)

- 구조체 기본 문법과 메모리 구조
- 구조체를 이용한 데이터 묶음 표현
- 설정 값(config)을 구조체로 관리하는 이유
- MCU 코드에서 구조체 기반 설정 방식의 장점

> 주변장치 초기화 코드에서는 여러 설정 값을 하나의 구조체로 관리하는 방식이 자주 사용된다.

---

### 5. 포인터와 구조체의 결합

- 구조체 포인터 접근 (`→` 연산자)
- 구조체 배열
- 함수 인자로 구조체 포인터를 전달하는 패턴

> 실제 MCU 실습 코드에서는 **구조체 + 포인터 조합**이 가장 빈번하게 등장하는 형태 중 하나이다.

---

## 정리

`00_Basic`은 **C 언어 자체를 학습하기 위한 공간이 아니라**,

- MCU 실습 코드가 왜 이런 형태로 작성되는지
- 헤더 파일과 소스 파일이 어떤 역할을 나누어 가지는지
- 설정 코드가 하드웨어 동작과 어떻게 연결되는지

를 이해하기 위한 **기초 체력 구간**이다.

이 디렉터리의 내용을 충분히 이해한 후에 `01_GPIO` 이후의 실습을 진행하면, 단순한 코드 따라치기가 아닌 **의도 있는 코드 해석과 수정**이 가능해진다.
